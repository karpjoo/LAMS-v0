# Open Questions

프로젝트 설계 및 구현 방향에 영향을 미치는 미해결 질문 및 해결 완료 사항의 목록입니다.

## 1. Open Questions (미해결 질문)
- **Q-105**: 관리자(Admin)가 평가자의 라벨링 진행 상황을 모니터링할 때, 어떤 평가자가 어떤 대화(call_id)에 라벨링을 안 했는지 개별 세부 진척 현황(상담원별 완료 건수 등)을 조회할 수 있게 해야 하나요?
  - *도메인 영향*: Dashboard 컴포넌트 내에서 개별 Evaluator 진척 통계 레코드 쿼리 범위 및 개인정보 격리 수준 정의에 영향을 미침.
- **Q-201**: 상담 대화 데이터셋과 외부에서 생성된 LLM 판정 결과(JSON) 및 외부에서 산출된 통계 데이터의 필드 상세 명세서 정의.
  - *도메인 영향*: 파일 업로드 파서 모듈의 필드 유효성 검사 로직(Stage 6 Data Design 단계에서 구체적으로 확정될 예정)을 가로막고 있음.

## 2. Resolved Questions (해결 완료 질문)
- **Q-101**: 일반 상담원 그룹과 언어학 전문가 그룹이 라벨을 남길 때, 서로 다른 화면이나 제약 사항이 존재하나요?
  - *해결*: 두 그룹은 별도의 전용 화면 분리 없이 동일한 공용 평가 화면을 공유하며 동일한 평가 권한(`ROLE-002 Evaluator`) 수준으로 통일하여 도메인을 단순화하기로 결정함.
- **Q-102**: 전문가 합의 라벨(Gold Standard)을 구성하는 다수결(Adjudication) 프로세스를 웹 UI에서 직접 지원해야 하나요?
  - *해결*: 본 LAMS-v0 범위 내의 웹 도메인 복잡성을 낮추기 위해, 직접적인 다수결 판정 기능은 제공하지 않으며 외부 사전 통계 업로드에만 집중함.
- **Q-103**: 관리자가 데이터셋을 추가 업로드할 때, 기존 평가가 완료된 상담 대화와 라벨 데이터를 유지한 채 데이터만 병합(Upsert)해야 하는가, 아니면 덮어쓰기(Overwrite) 형태로 관리해야 하는가?
  - *해결*: 기존 평가 결과의 소실 위험을 원천 방지하기 위해 덮어쓰기를 제한하며 멱등성을 지키는 병합 방식(`BR-006`)을 도메인 규칙으로 고정함.
- **Q-104**: 로그 데이터(AUD-001, AUD-002)를 Firestore 내 별도 로그 컬렉션에 저장하는 것으로 충분한가요?
  - *해결*: Firestore 내의 별도 `audit_logs` 컬렉션을 생성하고, Security Rules를 통해 편집/삭제를 전면 금지하는 Append-only 규칙(`BR-004`, `INV-003`)을 적용하여 충족하기로 결정함.
