# LAMS-v0 사용자 매뉴얼 (User Manual)

LAMS-v0(LLM & Agentic Multi-evaluator System)은 콜센터 유해 발화 탐지 연구의 신뢰도를 높이기 위해 설계된 **다중 평가자 비교 분석 및 합의 지원 시스템**입니다. 본 매뉴얼은 역할별 권한 구조 및 주요 기능의 상세한 사용 단계를 안내합니다.

---

## 1. 역할군 및 권한 개요 (System Roles)

LAMS-v0는 세 가지 사용자 역할을 지원하며, 각 역할에 따라 대시보드와 워크스페이스의 접근 및 제어 수준이 달라집니다.

1.  **관리자 (Admin)**: 시스템의 데이터 주입 및 관리, 프로세스 설정 제어, 전체 감사 로그 모니터링을 담당합니다.
2.  **평가자 (Evaluator)**: 개별 상담 대화를 검토하여 유해 발화 여부, 위해 레벨, 구체적인 텍스트 증거를 라벨링합니다.
3.  **합의조정자 (Adjudicator)**: 평가자 간 라벨 의견이 불일치하는 충돌(Conflict) 상황에서 최종 조율(Gold Standard) 결정을 내립니다.

---

## 2. 관리자 (Admin) 매뉴얼

관리자 관리 대시보드는 [http://localhost:5173/admin](http://localhost:5173/admin) 경로(프로덕션의 경우 해당 도메인의 `/admin` 경로)를 통해 접속할 수 있습니다.

### 2.1 대화 데이터셋 업로드 (CSV/JSON)
1.  **대화 데이터셋 업로드 카드**에서 파일 브라우저 버튼을 누르거나 준비된 데이터셋 JSON 파일을 드래그 앤 드롭합니다.
2.  선택한 파일의 스키마 및 내용 유효성 검사가 자동으로 수행됩니다.
3.  **개인정보 경고 스캔 (PII Warning Scan)**:
    *   업로드된 데이터 내부에 실명, 전화번호 등 개인정보 패턴이 정규식(Regex) 매칭을 통해 검출될 경우 경고 팝업이 노출됩니다.
    *   **취소(Cancel)** 또는 경고를 확인 후 **승인(Confirm)**을 선택해 다음 단계로 진행할 수 있습니다.
4.  **Commit Upload** 버튼을 누르면 Firebase Firestore 데이터베이스로 트랜잭션이 수행되며 전체 로그에 기록됩니다.

### 2.2 프로세스 설정 (Process Settings)
*   **2차 평가 생략 및 즉시 3차 이송 (Skip Round 2 on Mismatch)** 체크박스:
    *   이 옵션을 활성화하면 1차 평가(Round 1)에서 평가자 간 불일치(Conflict)가 생길 경우, 2차 평가(Round 2)를 생략하고 즉시 합의조정자 조율 단계(Round 3 Active)로 대화 상태가 전환됩니다.

### 2.3 외부 LLM 예측 결과 및 통계 데이터 업로드
*   **Import LLM Predictions 카드**:
    *   오프라인에서 사전 생성된 LLM(대형 언어 모델)의 예측 라벨 결과 JSON 파일을 업로드하여 데이터베이스의 `/llm_results` 컬렉션을 갱신합니다.
    *   *주의: 데이터셋 업로드가 선행되어 존재하는 `call_id`인 경우에만 커밋이 완료됩니다.*
*   **Import Agreement Statistics 카드**:
    *   오프라인 통계 엔진에서 사전 계산된 다중 평가자 간의 신뢰도 지표인 Fleiss' Kappa 및 Cohen's Kappa, F1 스코어 등이 정의된 통계 JSON 파일을 업로드하여 대시보드 지표를 동기화합니다.

### 2.4 데이터베이스 초기화 (Danger Zone)
1.  **Clear Database Records** 버튼을 클릭합니다.
2.  실수 방지를 위한 안전 장치로 텍스트 입력창 모달이 노출됩니다.
3.  텍스트 필드에 정확히 영문 대문자로 **`"DELETE ALL DATASET"`**을 직접 타이핑해야만 삭제 승인 버튼이 활성화됩니다.
4.  삭제 시 대화 데이터, 평가 라벨, LLM 예측 및 골드 스탠다드 데이터가 **영구 삭제**되며, Audit Log에 이력이 누적됩니다.

## 3. 평가자 (Evaluator) 매뉴얼

평가자는 로그인 후 평가 워크스페이스인 `/workspace` 화면을 사용하게 됩니다.

### 3.1 대화 목록 탐색 (Conversations Sidebar)
*   좌측 사이드바에서 현재 시스템에 주입된 대화의 `call_id` 목록을 확인할 수 있습니다.
*   각 대화 항목 하단에는 **라벨 작성 완료 개수 (Labels: X/3)**와 **현재 라벨 프로세스 진행 상태**가 라벨로 표시됩니다.
    *   `Round 1 Active`: 1차 평가자들의 초기 독자 평가가 진행 중인 상태.
    *   `Round 2 Active`: 1차 불일치에 의해 2차로 상세 검토 및 평가를 진행 중인 상태.
    *   `Round 3 Active`: 1, 2차 결과의 불일치로 합의조정자가 조정 최종 결정을 내리는 단계.
    *   `Consensus Reached`: 의견 일치가 이뤄졌거나 최종 합의 라벨(Gold Standard)이 지정되어 평가가 락(Lock)된 상태.
*   사이드바 최상단의 검색 기능을 통해 특정 `call_id` 대화를 빠르게 필터링할 수 있습니다.

### 3.2 대화 턴 검토 (Timeline turns Stream)
*   가운데 영역에는 선택된 상담 대화 내용이 상담사(Agent)와 고객(Customer) 간의 타임라인 형태 말풍선 스트림으로 정렬되어 보여집니다.
*   평가자는 대화 흐름을 순차적으로 읽으면서 유해하거나 협박성, 성희롱 등 문제 소지가 있는 발화를 심층 파악합니다.

### 3.3 평가 양식 작성 및 제출 (Toxicity Label Entry)
우측 패널의 양식(Evaluation Form)을 통해 유해 발화 탐지 평가를 작성합니다.

1.  **유해성 유무 (Toxicity Presence)**: 유해 발화 존재 여부 (`Present` / `Absent`)를 결정합니다.
2.  **유해 카테고리 (Toxicity Category)**: 폭언, 성희롱, 협박, 악성 민원 등 문제 발화의 상세 범주를 다중 선택 또는 지정합니다.
3.  **위험도 레벨 (Risk Level)**: 발화의 수위와 피해 심각성에 따라 위해 등급(`Low`, `Medium`, `High`)을 선택합니다.
4.  **핵심 증거 문구 발췌 (Evidence Phrase Excerpt)**:
    *   대화 내에서 판정의 결정적 근거가 되는 단어나 구절(Evidence Phrase)을 입력합니다. (여러 개 등록 가능)
5.  **제출 (Save Evaluation)**:
    *   양식을 모두 채우고 제출하면 데이터가 Firestore에 반영됩니다.
    *   **Blind-spot 정보 격리 보안 규칙**: `Round 1 Active` 상태에서는 타인의 독립적인 판단을 해치지 않기 위해 다른 평가자들이 남긴 평가 내역을 볼 수 없도록 데이터베이스 보안 규칙이 설정되어 격리됩니다.

## 4. 합의조정자 (Adjudicator) 매뉴얼

합의조정자는 의견 충돌이 발생하여 `Round 3 Active`로 이송된 대화들을 모아 최종 합의 판정을 도출합니다.

### 4.1 불일치 내역 확인 및 대조
1.  사이드바에서 `Round 3 Active` 상태의 대화를 선택합니다.
2.  우측 패널에 **"이전 차수 평가 결과 (Other Evaluations)"** 영역이 활성화됩니다.
3.  이곳에서 1~2차 평가자들이 입력했던 유해성 유무 판단, 유해 카테고리 정보, 위해 등급과 각 평가자들이 근거로 지적했던 **핵심 증거 문구(Evidence Phrases)**들을 한 화면에서 상세히 대조 및 비교해 봅니다.

### 4.2 최종 합의 라벨 (Gold Standard) 등록 및 프로세스 마감
1.  평가자 간 의견과 증거 대조 결과를 충분히 검토한 뒤, 최종 조율 결정을 우측의 **Toxicity Label** 폼에 입력합니다.
2.  제출을 완료하면 시스템은 입력된 데이터를 바탕으로 **골드 스탠다드(Gold Standard) 최종 승인본** 데이터를 기록합니다.
3.  성공적으로 커밋되면 해당 대화의 상태는 즉시 **`Consensus Reached`**로 전이되며, 추가적인 평가 수정 및 제출 기능은 데이터 보호를 위해 완전 차단(Lock)됩니다.

---

## 5. 대시보드 (Dashboard) 활용 가이드

모든 사용자는 `/dashboard` 경로에서 다중 평가자 간의 합의도 통계를 조회할 수 있습니다.

*   **동의율 모니터링**: Cohen's Kappa, Fleiss' Kappa 수치를 통해 인간 평가자 간의 평가 정밀도 수준을 시각적으로 파악합니다.
*   **LLM 성능 비교 지표**: 사람이 도출한 최종 골드 스탠다드 대비 기계 학습 모델(LLM)의 예측 정확도 및 F1 Score 통계를 실시간으로 비교 검토하여 모델 고도화에 기여합니다.
