# 02 Admin Power Review

## 1. Privileged Actors
- **ROLE-001 (Admin)**: 연구를 지휘하고 전체 데이터를 관리할 수 있는 유일한 권한자.

## 2. Privileged Powers
| Power / Action | Intended Purpose | Security / Privacy Risk |
|---|---|---|
| 대화 데이터셋 일괄 등록 (Import) | 연구용 실제/합성 대화 데이터를 Firestore 데이터베이스에 로드 | 악성 스키마 업로드로 인한 시스템 장애 또는 비식별화 누락 데이터 유입 |
| 대화 데이터셋 전체 삭제 (Delete) | 연구 종료 혹은 오작동으로 인한 데이터 초기화 | 실수에 의한 연구 진행 데이터 영구 소실 |
| 외부 LLM 판정 결과 일괄 업로드 | LLM 성능 평가 대조 데이터 축적 | 형식 에러가 있는 파일 입력 시 매핑 실패 및 UI 에러 |
| 외부 Kappa 통계 결과 일괄 업로드 | 다수 평가자 일치도 결과 데이터 반영 | 통계 위변조 및 계산값 오염 |

## 3. Risk
관리자 권한의 탈취 혹은 조작 실수는 본 플랫폼의 가용성과 데이터 무결성을 파괴하는 최대 리스크 요인입니다. 또한 악성 유저가 관리자 UI 경로(`/admin` 등)를 추정하여 직접 접근할 우려가 있습니다.

## 4. Reversibility
- **일괄 업로드**: 신규 업로드 시 기존 데이터를 덮어쓰거나 수정할 수 있어 복구성이 비교적 높습니다.
- **일괄 삭제**: Firestore에서 삭제된 문서는 기본적으로 백업이 없을 시 즉각적이고 영구적으로 유실되어 되돌릴 수 없습니다. (Irreversible)

## 5. Audit Required
- 관리자(ROLE-001)의 모든 일괄 삭제(Delete) 작업 및 신규 데이터셋 업로드(Import) 시점은 시스템 내부 로그 테이블(`AUD-001`)에 행위자 ID, 타임스탬프와 함께 로깅되어야 합니다.

## 6. Recommended Constraints
- **UI 경고 인터락**: 전체 대화 셋 삭제 액션 실행 시, 확인 팝업 모달창에 "DELETE ALL DATASET"을 정확히 입력해야만 버튼이 활성화되도록 유도합니다.
- **서버사이드 권한 검증**: 웹 프론트엔드 라우팅 제어 외에도 Firestore Security Rules에서 관리자 Uid 목록 또는 특정 어트리뷰트가 일치하는 사용자만 쓰기 작업을 할 수 있게 데이터 레벨에서 강력히 보호합니다.
